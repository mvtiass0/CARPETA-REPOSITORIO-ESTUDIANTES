# pilares: descomposición, reconcimiento de patrones, la abstracción, algoritmos. 

## Descomposición:
dividir para conquistar, un problema complejo y grande, se rompe en partes pequeñas y manejables. se hace de a poco en procesos para llevarlo de manera más liviana, se descompone en diseño A, B, C y códigos en las funciones propias, no solo se pone todo en draw, si no se ponen difrentes 
titulos para ordenar. 

## recocimiento de patrones:

si algo se repite o sigue una lógica constante se puede optimatizar.
....

## Abstracción: 

lo importante vs el detalle, filtrar la información que es innecesaria, solo definir las caracteristicas que definen el problema y crear una representación simbolica, se usa a traves de variables y map

## Algoritmos: 

receta de paso a paso, plan de acción que debe seguir el sistema, a traves de reglas ordenadas. Se traduce en el diagrama de flujo y condicionales (if/else), mapa logico que conecta todas las partes anteriores. 

# Tipos de interacción: 

Interacción discreta: evento especifico, es un interruptor de encendido y apagado, como mousePressed o if(mouselsPressed

interacción continua: cuabdo un sistema reacciona constanrtemente un movimiento
